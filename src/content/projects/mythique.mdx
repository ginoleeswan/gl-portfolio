decisions:
  - icon: "lucide:layers"
    title: One codebase, three platforms
    body: Expo's managed workflow ships the same file-based routes to iOS, Android, and web. The trade is living inside the native-module boundary; the win is a single codebase to maintain.
  - icon: "lucide:database"
    title: Supabase in front of the APIs
    body: Two rate-limited comic APIs sit behind Supabase, so reads stay fast, the third-party keys stay server-side, and the UI only ever sees one normalized shape.
challenge: Two APIs with different shapes, gaps, and rate limits had to look like one clean dataset — missing fields degrade gracefully instead of breaking the screen.
